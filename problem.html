# 视频监控

#

专网对接政府平台而不能外网对接建造平台

!> 所属分类：对接冲突

涉及项目：兰泰项目(外地)

详情：由于当地政府要求视频监控设备必须通过视频专网对接到政府平台，而未布设普通外网以及额外的设备与平台进行对接。

解决方法：通过使用外网以及部署一套NVR设备即可与平台对接（领导要求观看监控时，联系分包商做切换）。

# 人员两制

#

闸机设备无法联网且无法对接平台或云筑网（闸机不符合标准）

!> 所属分类：设备问题、沟通问题

涉及项目：兰泰项目(外地)

详情：项目最初使用的设备无法联网，后续反馈更换设备后可对接云筑网，但设备迟迟未到场，多次推进后又反馈到场的设备无法对接云筑网。项目极不配合且敷衍。

解决方法：闸机由分公司进行统一集采。对于不配合且敷衍对个人进行扣分，停工资，罚款处理。

#

闸机设备通过云筑网对接的数据传输不稳定

!> 所属分类：对接方式

涉及项目：深汕二期

详情：早期的闸机安装出现过较多问题，一开始购买的闸机不符合标准没有数据传过来，后和甲方沟通后，闸机供应商对闸机经过处理可以和云筑网对接，但数据传输不够稳定。

解决方法：闸机由分公司进行统一集采。闸机数据接口直接对接两制系统。

# 安全生产

#

现场安全员整体配合度较低

!> 所属分类：态度问题

涉及项目：深汕二期

详情：现场安全员整体配合度较低，需要通过反复沟通和扣分才能推动一点，同时安全员本人也反馈由于需要使用的安全记录APP太多，工作量太大。

解决方法：按时只能督促他每周上传安全记录。

# 飞行航拍

#

高建筑环境导致无法自动航飞无人机

!> 所属分类：环境影响

涉及项目：高新园

详情：因项目周围大型建筑物比较密集，环境较为复杂，项目无法进行无人机建模和预设航线自动巡检，只能手动飞行拍照，由于是环境客观因素导致，暂无解决办法。

解决方法：若项目存在同样问题，为保障无人机飞行安全，要项目与领导协调沟通申请只进行图片拍摄，不进行无人机建模。

#

磁场干扰导致无法无人机出现失控现象或炸机

!> 所属分类：环境影响

涉及项目：笋岗

详情：与项目了解，项目周围环境磁场干扰严重，无人机飞行时经常出现失控现象，然后导致炸机，炸机后没有重新采购无人机，无法进行飞行图片的上传。

解决方法：若出现这种情况，需要与领导申请，若领导同意项目不用采购无人机，此项内容可不进行考核。

#

无人机型号不支持导致前期无法对应到司空平台，导致无人机对接落后

!> 所属分类：设备问题、沟通问题

涉及项目：新能源

详情：项目因紧急使用，自己购买了精灵4 advance , 不支持司空对接、购买无人机型号未前期沟通清楚。项目不愿多花钱买第二台无人机。

解决方法：汇报确定项目是否必需要有无人机飞行数据，新能源项目为重点关注项目，必须要有飞行数据对接，其余项目可协商是否只上传飞行照片。分子公司统一采购无人机，减少项目使用新飞机成本。项目采购无人机与科创中心提前确认。

# 模型及图纸

#

模型输出不及时

!> 所属分类：沟通问题

涉及项目：长腰岭

详情：项目部分深化图纸完成后直接对接到工厂进行构件的生产，未及时同步到BIM所。

解决方法：项目宣贯是确保项目人员、深化所、BIM所、工厂相关责任人均了解构件追溯各时间节点各自的工作任务和职责，即使了解构件生产出厂计划。

#

模型不符合追溯要求，有构件遗漏

!> 所属分类：沟通问题

涉及项目：二十高

详情：收到的模型是预制构件+现浇部分，构件变更，有构件遗漏，图纸与模型不吻合，构件重叠导致生成列表时数量不对，模型和图纸中的构件名称不吻合。

解决方法：一定要跟BIM所负责该项目模型的同事强调模型标准，拿到模型，一定要与图纸进行比对，生成二维码之前，要仔细核对构件数量，确保与图纸、模型一致，与BIM所、项目要保持沟通，一旦模型有变更，要尽快知晓，并完成二维码的更新。

#

模型版本更新，但没有第一时间收到设计变更通知单及修改后的模型

!> 所属分类：沟通问题、图纸问题

涉及项目：笋岗

详情：模型版本更新，但没有第一时间收到设计变更通知单及修改后的模型。

解决方法：若设计变更影响构件追溯，则发现后及时跟进模型修改进度，督促BIM所完成模型的更新，然后尽快生成二维码给到工厂和项目，并将模型更新到数字设计模块若设计变更不影响构件追溯，发现后与BIM所联系确定模型修改的完成时间，届时再找BIM所，然后将模型更新到平台数字设计模块。

#

钢结构公司发现二维码无法识别

!> 所属分类：沟通问题、图纸问题

涉及项目：新能源

详情：钢结构公司深化图纸与设计院图纸存在很大差异，依据设计院图纸建模生成的二维码不可用。

解决方法：以最终定稿的深化图纸为准进行BIM建模。

#

钢结构公司深化进度不会全部出完，只比实际进度快了一层

!> 所属分类：沟通问题、分包问题

涉及项目：新能源

详情：钢结构公司深化进度不会全部出完，只比实际进度快了一层。

解决方法：BIM所基于深化图纸逐层建模 逐层生成二维码。

#

钢构模型输出慢，钢构为外包，扫码消极

!> 所属分类：沟通问题、分包问题

涉及项目：徐州园博园

详情：钢构模型输出慢，钢构为外包，扫码消极。

解决方法：与项目总工沟通或领导层面进行施压落实，同时进行扣分处理。

#

图纸定稿迟，构件类型多，模型生成迟。图纸经常更新

!> 所属分类：沟通问题

涉及项目：龙岗三校

详情：二维码生成延迟，版本更新4次，人手对比生成的构件编号有漏洞易出错，二维码混乱，有重复编号二维码。

解决方法：二期上线改善生成二维码的问题；图纸定稿迟、经常更新时，先做定稿的地下部分的模型，地上的模型的根据现场进度和图纸状态逐步更新。

# 构件追溯

#

扫码不够及时

!> 所属分类：态度问题

涉及项目：龙岗三校

详情：扫码不够及时。

解决方法：前期开会时需要拉上重要领导，让领导清楚信息化重要性。找项目人员要周报，根据实际进度催扫码。

#

钢结构分包商不配合，二局安装（构件追溯重点注意对象）

!> 所属分类：态度问题、分包问题

涉及项目：龙岗三校

详情：对分包没有管理手段。

解决方法：找项目经理告状，天天在有领导的群里催扫码，用管理办法项目考核、工程例会等方法轮流催。

#

扫码工作堆积问题

!> 所属分类：态度问题、沟通问题

涉及项目：二十高

详情：扫码工作堆积问题。

解决方法：及时进行补扫工作，前期提前做好扫码培训工作并落实到个人。

#

分包对构件追溯流程不清楚，构件追溯开展不及时

!> 所属分类：分包问题、沟通问题

涉及项目：生物医药

详情：分包对构件追溯流程不清楚，构件追溯开展不及时。

解决方法：构件追溯开展一定要约上领导、各参与方开会，多次开会，落实每个环节的负责人，记录会议纪要。

#

钢结构构件追溯开展存在模型不可用，反复多次修改发布二维码

!> 所属分类：沟通问题

涉及项目：新能源

详情：钢结构公司不了解构件追溯的本质逻辑要求，未在前期提出基于深化图纸建模。

解决方法：BIM所基于深化图纸逐层建模 逐层生成二维码。

# 需求沟通

#

项目领导不重视、不积极配合

!> 所属分类：态度问题

涉及项目：兰泰、深汕二期

详情：项目从项目经理到信息化负责人再到各模块责任人对智慧建造平台信息化部署不重视，配合敷衍，扣分也未改变。

解决方法：扣工资。

#

在数据对接上，存在技术、沟通协调、商务等多方因素，导致对接效率会很低

!> 所属分类：对接问题

涉及项目：长腰岭

详情：外地项目，未经过公司统一集采，在数据对接上，存在技术、沟通协调、商务等多方因素，导致对接效率会很低。

解决方法：实名分账平台扩展可对接设备及平台种类，提升平台的通用性，同时对于新合作分包商需提前部署数据对接工作。

#

整体的配合度不高

!> 所属分类：态度问题

涉及项目：深汕二期

详情：深汕厂在开始开始信息化管理之前已经完成了主体结构等大部分工程，因此没有构件、模型、商务等数据。

解决方法：暂无。

#

开会信息化负责人和领导基本不在

!> 所属分类：沟通问题

涉及项目：生物医药

详情：项目大，信息化负责人是总工，较忙。

解决方法：落实各信息化版块负责人。

#

采购流程复杂，现场环境恶劣

!> 所属分类：环境影响

涉及项目：徐州园博园

详情：徐州项目无人机需要华东公司审批，政府平台不允许部分闸机供应商对接，监控安装受大雪大风影响。

解决方法：与华东信息化相关人沟通推进采购进度，更换品茗闸机设备，天气良好时监控统一进行安装调试。

#

人员架构调整，项目复杂，涉及人员多，沟通困难

!> 所属分类：沟通影响

涉及项目：徐州园博园

详情：两次更换项目人员通讯录，BIM负责人离职，信息化负责人已经是第三个。

解决方法：点对点落实到各模块负责人电话沟通，各群在集体交流并形成会议纪要，落实到个人。
